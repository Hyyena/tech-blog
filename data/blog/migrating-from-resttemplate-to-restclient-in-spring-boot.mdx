## Introduction

Spring Boot 애플리케이션에서 외부 서비스에 HTTP 요청을 구현하기 위해 Spring이 제공하는 `RestTemplate`, `WebClient`와 같은 HTTP 클라이언트가 사용됩니다.

`RestTemplate`은 Spring 3.0 버전부터 도입되었고, 템플릿 메소드 API를 사용하는 동기식 클라이언트로 전통적으로 많이 사용되어 왔습니다.

`WebClient`는 Sporing 5.0 버전에 추가되었고, 모던 API 형식과 더불어 함수형 프로그래밍이 가능해 `RestTemplate` 대비 사용이 편리하고 직관적입니다.

하지만, `WebClient`를 사용하기 위해서는 `WebFlux (spring-boot-starter-webflux)` 의존성이 필요합니다. `WebFlux`는 비동기나 스트리밍 데이터 처리와 같이 리액티브 프로그래밍을 위해 필요한 프레임워크이기 때문에,
동기 방식의 HTTP 구현을 위한 애플리케이션에서 불필요한 의존성이 추가되는 단점이 존재합니다.

이러한 상황에서 Spring Boot 3.2 (Spring 6.1) 부터 추가된 `RestClient`를 사용하면 기존 `MVC (spring-boot-starter-web)` 애플리케이션에서 별도의 의존성 추가 없이 `WebClient`와 유사한 방식으로 모던한 방식으로 동기식 HTTP 클라이언트를 구현할 수 있습니다.
